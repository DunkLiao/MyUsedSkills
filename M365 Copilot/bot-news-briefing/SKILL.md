# 臺銀新聞一頁式簡報（BOT News Briefing）

把一份新聞彙整檔（多家媒體、多則新聞）整理成只談臺灣銀行的高層簡報。

## 觸發時機（Trigger）

當使用者提供新聞彙整／剪報／新聞全文，並要求：
- "篩選臺銀相關新聞"、"整理臺銀新聞"、"臺灣銀行相關新聞摘要"
- "做成一頁式簡報"、"給高層的臺銀新聞摘要"、"整理成 Word 純文字"

## When NOT to Use

- 與臺銀無關的一般新聞摘要 — 直接回答即可。
- 使用者要求產出 .docx / .pdf 實體檔案 — 改用 docx / pdf 技能（本技能只輸出可複製純文字）。
- 要求寄送 email 或 Teams 訊息 — 改用對應的寄送工具。
- 市場風險、保險業整體、其他銀行為主體之新聞（除非臺銀為主要當事人）。

## 篩選範圍：何謂「臺銀相關」

納入（臺銀為主要當事人）：
- 臺灣銀行／臺銀本身的業務、商品（如黃金存摺）、獎項、人事、裁罰、內控、洗錢防制。
- 臺銀人壽（臺灣金控集團子公司，源自臺灣銀行分割）之增資、虧損、投資布局、法遵等。
- 臺銀在產業／政策新聞中明確為「主要主體」者（例：放款餘額居首、最積極承作之銀行）。

排除（臺銀僅被附帶提及）：
- 臺銀僅為聯貸案參貸行之一、論壇出席名單之一、被引述的市場觀察來源。
- 以央行、金管會、其他銀行或整體產業為主體，臺銀只出現一兩句者。

判斷不確定時，以「臺銀是否為該則新聞的核心當事人」為準；否則排除。

## 工作流程（Workflow）

### Step 0：取得來源
1. 若使用者附加檔案，直接以 **Read** 讀取（檔案常於 `input/`）。
2. 檔案常很大，先用 **Grep** 搜尋 `臺銀|台銀|臺灣銀行|台灣銀行|臺銀人壽|BOT`（加 `-n`、`-C 2`
   取得行號與前後文）定位相關段落，再用 **Read** 帶 `offset`/`limit` 讀取對應行區間，
   避免一次讀爆。
3. 用 **Grep** 找出每則的來源網址（`https?://`）與媒體、發佈日期，對應到各篇全文段落。

### Step 1：篩選並彙整為「結構化分述」（中間產物）
1. 依上方「篩選範圍」挑出臺銀相關新聞。
2. **合併同一事件的多家報導**：同一則故事（如同一獎項、同一場質詢）即使有 5~6 篇不同
   媒體版本，也整併為「一則」，避免高層閱讀重複內容。
3. 每則故事整理：標題、總述（2–3 句）、分述（條列細節）、總結（新聞意義）、關鍵字
   （3–5 個）、備註（背景＋對臺銀的關聯性／影響性）。
4. 數字若各家報導不一致，並列並註明出處媒體，不自行取捨或新增。

### Step 2：產出「一頁式 Word 純文字高層簡報」（最終成果）
將 Step 1 內容濃縮成一頁、可直接複製貼入 Word 的純文字：
1. 以純文字呈現，整段置於單一程式碼區塊（fenced code block），方便一鍵複製。
2. **移除所有超連結／網址。**
3. **重要數據整理成純文字表格**（用 `┌─┬─┐│├┤└┴┘` 框線，欄位以全形空白對齊，貼入
   Word 等寬字型時不跑版）。文字敘述保留精簡重點，數字盡量進表格。
4. 每則：小標題 → 1~3 句敘述 → 數據表（若有）→「意義：」一句點出對臺銀的影響。
5. 控制在約一頁篇幅；敘述從簡，讓表格承載數字。

## 輸出格式（Output Format）

最終回覆**只輸出簡報本身**，整段放入一個程式碼區塊，結構如下（範例骨架）：

```
臺灣銀行相關新聞摘要　　報告日期：YYYY/MM/DD　　（一頁式・高層簡報）
═══════════════════════════════════════════════════════════════

一、<故事標題>
　<總述 1–3 句>
　┌──────────┬──────────────────────────┐
　│ 項目　　 │ 數據　　　　　　　　　　 │
　├──────────┼──────────────────────────┤
　│ …　　　　│ …　　　　　　　　　　　 │
　└──────────┴──────────────────────────┘
　意義：<對臺銀的關聯性或影響性，一句>

二、<下一則>…
═══════════════════════════════════════════════════════════════
```

若使用者另要求「結構化分述」版本（Step 1），則每則依序輸出：標題表頭、總述、分述、
總結、關鍵字、備註（此版本可保留網址；一頁式 Word 版本則不放網址）。

## 失敗處理（Failure Handling）

- 檔案過大無法整檔讀取：改用 Grep 定位 + Read 分段讀取，切勿略過內容。
- 檔案讀不出（毀損／格式不支援）：照實告知，請使用者重新提供，**不得以範例或假資料替代**。
- 來源中找不到任何臺銀相關新聞：照實說明「來源無臺銀相關內容」，不硬湊。
- 數值需加總或換算時：用程式（Bash/python）計算後再填入，不得心算（本技能原則上只搬運
  來源既有數字，不另行推算新數據）。

## Guardrails

- Never fabricate：不得新增來源外資訊；不杜撰名稱、數字、日期、引述或網址。
- Always 忠於來源數字：各家報導數字不一致時並列並註明媒體，不擅自合併或取平均。
- Never 加前言或 meta：最終輸出禁止開場白（如「以下為整理…」）、結語或無關文字。
- Never 在一頁式 Word 簡報內放連結／網址。
- Always 直接交付：來源足以整理時不提問；確實缺關鍵資訊才簡短詢問一次。
- Never 評論個別員工績效：涉及裁罰／內控新聞僅陳述來源事實，不對個人作績效評價。
- Always 只輸出純文字：本技能不產生 .docx/.pdf 檔；需實體檔案請改用 docx/pdf 技能。
