# 臺銀月報 → 定期定額基金規劃 Excel

把上傳的臺灣銀行「投資組合建議月報」PDF，轉成一份**可重複使用、資料驅動**的定期定額規劃
Excel。流程＝從 PDF 萃取基金資料 → 依風險屬性與預算組規劃 → 用內附建檔器套臺銀配色＋圖表
產出 `.xlsx`。

## When NOT to Use

- 使用者要的是一般／非臺銀月報的基金建議，或要查即時行情 → 不適用本技能。
- 要把月報做成新聞彙整、Word、簡報 → 用 `bot-news-briefing` / `docx` / `pptx`。
- 要產出可重複套用的佈景主題 `.thmx` → 用 `bot-brand-theme`。
- 授信戶重大變故風險評估報告 → 用 `bot-credit-risk-report`。
- 寄送 email / Teams 訊息 → 用對應的郵件／Teams 工具。

## 必要輸入（缺哪個就先問一個問題，不要一次問一堆）

1. **月報 PDF**：臺銀投資組合建議月報（通常已附在 `input/`，先用 Glob 找）。
2. **風險承受度**：積極型／穩健型／保守型（決定納入的風險等級，見下表）。
3. **每月定期定額預算（NT$）**：例如 15,000。

風險屬性 → 納入之風險收益等級（以月報 RR1–RR5 定義為準）：

| 風險屬性 | 核心納入 | 可作衛星 |
|---|---|---|
| 積極型 | RR4、RR5（追求資本利得／最大資本利得、波動大） | RR4 高息債作衛星 |
| 穩健型 | RR3（兼顧資本利得與固定收益） | RR2、部分 RR4 |
| 保守型 | RR2（穩定收益為主） | RR1、RR3 |

## 工作流程

1. **找檔＋讀檔**：`Glob input/**/*.pdf` 找月報；用 `pypdf`（`PdfReader(...).extract_text()`）
   把全文存到 `working/`。月報結構通常含：投資人屬性配置建議、第二季精選基金（**定期定額推薦**／
   單筆推薦）、高齡客戶精選基金、本行保管推薦基金、理柏獲獎、銷售排名、以及
   **「本行架上績效表現穩定基金彙總表」**（這張表才有一年/三年數字）。

2. **篩選候選基金**（嚴格只取月報內標的）：
   - 取**符合該風險屬性風險等級**且**適合定期定額**者。
   - **務必同時納入兩類**：① 月報明列「定期定額推薦標的」者；② 未特別標註、但風險等級相符且
     屬波動較大之股票型（如保管推薦、單筆推薦中之股票型、彙總表內同屬性標的）者。
   - 月報對定期定額的選取邏輯是「**前景佳且波動性大、長期投資以分散平均成本效益**」——據此判斷。

3. **抓績效（近一年／近三年，原幣報酬率）**：優先從**彙總表**與各檔基金明細頁取數字。
   **精選頁的績效若為圖（無數字），且彙總表也查無 → 該欄填 `null`（會顯示「文件未提供」），
   絕不推估、絕不杜撰。** 若只查得到同基金**不同級別**（如日圓級別）數字，可採用並於 `note`
   註明級別差異。

4. **排序**：候選基金以**近三年績效（有數字者）由高至低**排序，無數字者列於其後。

5. **組三組投資組合**（每月預算 100% 配足，權重以小數表示且加總為 1.0）：
   - 三組需有**不同的基金搭配與配置邏輯**，且全部符合該風險屬性。
   - 範例邏輯（積極型）：① 鎖定績效最強標的；② 核心衛星（明列定期定額核心＋高息債衛星，
     最貼近月報該屬性建議比重）；③ 主題分散（以較低波動等級為主、區域更分散）。
   - 各檔給 `region`（台灣／中國／日本／南韓／美國／全球主題／新興市場／新興前緣債…）供比較頁用。

6. **寫規格 JSON**：依 `scripts/build_workbook.py` 檔頭 schema，寫到 `working/spec.json`
   （`funds`、`candidate_order`、`portfolios`、`monthly_budget`、`risk_profile`、`source`）。

7. **建檔（一律用內附建檔器，勿自行重寫產生器）**：
   ```bash
   python scripts/build_workbook.py working/spec.json "output/{風險屬性}定期定額投資組合建議_{年月}.xlsx"
   ```
   再用 xlsx 技能的重算腳本檢查零公式錯誤：
   ```bash
   python /opt/workspace-config/.claude/skills/xlsx/scripts/recalc.py "output/<檔名>.xlsx"
   ```
   建檔器會自動產出：說明頁、候選基金總表、**組合比較**（第3頁、橫向單頁、3圓餅＋2比較長條）、
   及各組合分頁（含圓餅圖、每月金額為 `=ROUND(預算*比例,0)` 公式）。

8. **交付前檢查（Delivery Gate）**：`Glob output/**/*.xlsx` 確認檔案存在；抽查各組合「合計」列
   為 **配置比例 100%、每月金額＝預算**；確認酒紅表頭與圖表都在。

## 使用的工具（Tools）

每個步驟用的具名工具如下，**務必使用內附建檔器這個 tool、不要自行重寫產生器**：

## 輸出格式（Output Format）

交付物：`output/{風險屬性}定期定額投資組合建議_{年月}.xlsx`（臺銀酒紅 #9A0036 配色、微軟正黑體）。
工作表固定六頁：說明、候選基金總表、組合比較、組合一／二／三。

回報格式：以 **bullet** 條列重點；候選基金與三組合在檔內均為 **table（表格）** 呈現，欄位
**grouped** 為〔基金名稱｜代碼｜風險等級｜近一年｜近三年｜配置比例｜每月金額｜特色〕。
用平實中文回報：檔名、分頁清單、三組合差異、以及「改說明頁預算數字可全檔連動」用法；
若有「文件未提供」欄位，主動說明哪幾檔需另查基金月報。不要貼出程式碼或內部工具名稱給使用者。

## Guardrails

- **Never fabricate** — 只用月報內的基金與數字，不得外搜或引用其他來源；不得杜撰名稱、代碼、績效、配息率。
- **績效缺值一律標「文件未提供」**，不以同類平均或市場常識回填（never invent 數字）。
- **失敗處理（failure handling）**：若 PDF 無法解析或找不到月報，**明確告知使用者並請其重新上傳，
  絕不以範例／示意／佔位數據替代真實內容**；若風險屬性或預算未提供，先 ask 一個問題再繼續。
- **Never auto-send / never post**：檔案只存 `output/`，不寄 email、不貼 Teams、不外流；review before delivery。
- **Always confirm** 風險屬性與每月預算後再產檔；三組合權重加總必須為 100%（不符建檔器會擋下）。
- **Privacy**：月報屬內部參考資料，回覆務必保留風險警語、do not 外流；不評論個別員工績效。
- **Always** 在交付前用 `recalc.py` 確認零公式錯誤、用 `Glob output/**/*.xlsx` 確認檔案存在。
- **配色固定**用臺銀色票（主色 `#9A0036`），不自創顏色、不產 `.thmx`（用 bot-brand-theme instead）。
- **金額用公式**（`=ROUND(預算*比例,0)`、`=SUM(...)`），不在 Python 端硬算後寫死。
- **配色固定**用臺銀色票（主色 `#9A0036`，輔色淺紅/金棕/金/深紫/灰藍），不自創顏色；不產 `.thmx`。
- 三組合權重加總必須為 100%（建檔器會擋下不符者）。
- 檔案只存 `output/`，不寄送、不外流；月報屬內部參考資料，回覆需保留風險警語。
- 不評論個別員工績效、不依受保護特徵分類；非屬本技能範疇者，轉介對應技能。
